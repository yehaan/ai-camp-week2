# outputs/gap-analysis.md — 초안 ↔ 골든 격차 분석

> outputs/draft-1.md (AI 첫 초안) vs golden/output-example.md (골든 = 학생 본인이 실제 작업·검수한 번역) 비교.
> 골든이 실제 운영 프로젝트 산출물이라, 격차가 곧 "AI 직역 ↔ 실무 번역"의 차이다.
> 이 격차가 3주차 "제약 조건"의 입력이 된다.

## 1. 빠진 정보 (골든엔 있는데 초안엔 없는 것 / 반대로 초안이 과하게 직역한 것)
- #8 「아직 신청한 상담이 없어요」: 초안은 「申し込んだ相談」까지 다 직역 → 골든은 「予約はありません」로 핵심만. 실무 번역은 화면 맥락상 불필요한 정보를 덜어냄
- #10: 골든은 「お悩み」「ご記入」 등 미화어로 정중도를 채움 → 초안엔 그 레이어가 빠짐

## 2. 구조 차이 (순서·형식·단어 선택이 다른 부분)
- #4 「회원 탈퇴 시」: 초안 「会員退会時」(직역) → 골든 「退会すると」(일본 UI 관용 구조)
- #4 「이전 내역」: 초안 「以前の内訳」 → 골든 「履歴」 (内訳=금액 명세 뉘앙스, UI에선 履歴)
- #7: 초안 「マイページに移動」 → 골든 「マイページへ移動」 + 읽기 쉬운 위치에 「、」 삽입

## 3. 톤 차이 (말투·격식·미화어가 다른 부분)
- #1 「申告する理由は何ですか？」(딱딱한 직역 의문문) → 골든 「通報理由を教えてください」(정중 요청형)
- #3 「ご参加ください」는 정중하나 #10 「作成してください」는 평범 → 초안은 정중도가 들쭉날쭉. 골든은 「ご記入ください」로 통일
- #9 「確定されました」(수동 직역) → 골든 「確定しました」(일본어 자연스러운 능동)

## 4. 사실 오류 / 추측 (직역으로 생긴 의미·일관성 오류)
- #3 「화상 상담」→ 초안 「ビデオ相談」: 일본 서비스 관용은 「オンライン相談」. 직역이 어색함
- #5 「비밀번호」→ 초안 「暗証番号」: **용어 일관성 붕괴**. 같은 시트 다른 화면에선 「パスワード」로 번역됨 — AI가 문맥 없이 행마다 다르게 옮김
- #2 「단순 변심」→ 초안 「単純な心変わり」: 일본 약관 관용어는 「自己都合」. 직역은 약관 문서로서 부적절

## 5. AI↔사람 경계 위반 (사람이 봐야 할 걸 AI가 단정 / 사람도 놓친 것)
- #1 申告↔通報, #3 ビデオ↔オンライン 같은 도메인 관용어를 AI가 검수 표시 없이 단정 — 의료·예약 도메인 지식이 필요한 사람 검수 영역
- #6 **골든(실제 번역)에도 오류 발견**: 원문은 「(선택)」인데 골든 일본어가 「(必須)」로 들어가 있음. 必須/任意 라벨은 AI든 사람이든 반드시 교차검수해야 한다는 증거
- 원본 시트에 미번역 행(「준비중」「검토 필요」)·내부 메모 혼입·**타사명(アニカネットワーク) 유출**이 있었음 — 사람 최종 점검 없이는 그대로 나갈 뻔함

## 6. 그 외 어색한 점
- 원문 자체에 오타 존재: 「확인할실」「작서해」「삭세」「지방읍입」 — AI가 자동 교정해버리면 원문 관리가 어려워짐. 골든은 사람이 의미로 보정함
- #9 등 placeholder({회원명}/{시간}/{병원명})를 초안이 그대로 두긴 했으나, 번역 대상으로 오인할 위험이 있음

## 7. 3주차에 줄 "제약 조건" 후보 (3개 이상)

1. **도메인 관용어 사전 적용** — 신고→通報, 화상 상담→オンライン相談, 단순 변심→自己都合, 회원 탈퇴→退会, 환불→返金(払い戻し 금지). 사전에 없는 용어는 직역 금지 + `⚠️ 사람 검수`
2. **용어 일관성 강제** — 같은 한국어 용어는 시트 전체에서 같은 일본어로. 특히 비밀번호→パスワード 통일(暗証番号 혼용 금지)
3. **경어·미화어 레벨 고정** — 안내·요청 문구는 정중 요청형(〜てください)·미화어(お〜/ご〜)로 통일, 수동 직역(確定されました) 금지
4. **必須/任意 라벨 절대 보존** — (필수)/(선택) 라벨은 추측·변경 금지, 원문 그대로. 출력 시 원문과 라벨 일치 자동 검증
5. **placeholder 보존 규칙** — {회원명}·{시간}·{병원명} 등 중괄호 변수는 번역 대상에서 제외, 위치만 유지
6. **원문 오타 처리 규칙** — 원문 오타 발견 시 자동 교정하지 말고 `⚠️ 원문 오타 의심` 표시 후 사람에게 전달
7. **대안 표현 의무 + 도메인 플래그** — 문구마다 채택안 1개 + 대안 2개, 도메인 관용어 불확실 시 검수 플래그

## 한 줄 회고

첫 초안에서 가장 크게 빗나간 한 가지는 **도메인 관용어 직역과 용어 일관성 붕괴**(申告/通報, ビデオ/オンライン, 비밀번호가 행마다 暗証番号·パスワード로 갈림) 였고,
그 이유는 (규칙을 안 줌 / 컨텍스트가 부족) 인 것 같다 — 도메인 용어 사전과 일관성 규칙을 컨텍스트로 주지 않아서 AI가 행 단위 직역기로 동작했다.
한편 골든(실제 번역)에도 「(선택)→(必須)」 오류가 있어서, **라벨·필수여부는 AI든 사람이든 반드시 교차검수가 필요하다**는 것도 같이 배웠다.
