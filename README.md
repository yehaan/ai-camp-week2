# AI CAMP 2주차 — 일본어 UX 라이팅·번역 보조 AI

> The Camp Week 2 · 한영은 (하네스 엔지니어링 평일반)
> 자동화 MVP 컨텍스트 세팅 + 첫 격차 분석

## 1. 내가 만들 자동화 MVP (한 줄)

**한국어 서비스 UX 문구 전반을 일본어로 현지화해주는 번역 보조 AI** — 버튼·라벨부터 안내문·약관·FAQ·푸시 알림까지, 글자수 제한·경어 레벨·업계 관용어·용어 일관성을 지킨 일본어 초안과 대안 표현을 생성한다. (도메인: 일본 뷰티·의료·시술 예약 앱)

## 2. AI가 맡을 일 / 사람이 최종 확인할 일

| AI가 맡는 일 | 사람이 최종 확인하는 일 |
|---|---|
| 한국어 원문 → 일본어 초안 생성 | 브랜드 톤·서비스 페르소나 적합성 |
| 자연스러운 대안 표현 2~3개 추천 | 뷰티·의료 업계 관용어의 적절성 |
| 글자수 제한·경어 레벨·용어 일관성 체크 | 법률·약관·필수/선택 라벨의 정확성 |
| 표기 흔들림·placeholder 보존 점검 | 실제 화면 맥락의 사용자 경험·릴리스 결정 |

**역할 경계 한 줄:** AI는 일본어 UX 마이크로카피 초안과 대안 표현을 생성하고, 사람은 브랜드 톤·업계 관용어·법률 표현·최종 릴리스 여부를 판단한다.

## 3. 사용한 입력 예시 / 이상적인 출력 예시 (요약)

**입력** (`golden/input-example.md`): 학생 본인이 실제로 진행한 일본 뷰티·의료 시술 예약 앱 현지화 프로젝트의 한국어 서비스 UX 문구 **100건**. **문구 유형 8개 카테고리**(안내·설명 / 마이크로카피·라벨 / 상태·피드백 / 약관·동의 / 취소·환불 정책 / 알림·푸시 / FAQ·Q&A / 에러·예외)로 분류 + 화면·톤 컨텍스트.

**이상적인 출력** (`golden/output-example.md`): 같은 프로젝트에서 학생 본인이 실제 작업·검수한 **한국어→일본어 번역 결과**. 제3자 창작이 아닌 실무 산출물이라 신뢰도가 가장 높다.

| 한국어 원문 | 일본어 (골든, 실제 번역) |
|---|---|
| 서비스 이용을 위해 로그인 해주세요. | サービス利用のため、ログインしてください |
| 신고하는 사유가 무엇인가요? | 通報理由を教えてください |
| 화상 상담에 참여해 주세요. | オンライン相談に参加してください |
| 단순 변심: 마음이 바뀌어 취소를 원하는 경우. | 自己都合：気が変わりキャンセルを希望する場合 |

## 4. 첫 초안과 골든 출력의 차이

AI 첫 초안(`outputs/draft-1.md`, 대표 10건)을 골든(학생 본인의 실제 번역)과 비교한 5가지 기준 (전체: `outputs/gap-analysis.md`):

| 기준 | 초안에서 빗나간 점 |
|---|---|
| 빠진 정보 | 불필요한 직역 (`신청한 상담` 전부 직역 vs 골든은 `予約`로 핵심만) |
| 구조 차이 | 일본 UI 관용 구조 무시 (`会員退会時` vs 골든 `退会すると`) |
| 톤 차이 | 정중도 들쭉날쭉, 수동 직역 (`確定されました` vs 골든 `確定しました`) |
| 사실 오류/추측 | 용어 일관성 붕괴 (비밀번호가 행마다 `暗証番号`·`パスワード`로 갈림), 도메인어 직역 (`ビデオ相談` vs `オンライン相談`) |
| AI↔사람 경계 위반 | 도메인 관용어를 검수 없이 단정. **골든에도 `(선택)→(必須)` 오류** 발견 — 라벨은 교차검수 필수 |

## 5. 3주차에 적용할 제약 조건 후보

1. **도메인 관용어 사전 적용** — 신고→通報, 화상 상담→オンライン相談, 단순 변심→自己都合 등. 미등록 용어는 직역 금지 + `⚠️ 사람 검수`
2. **용어 일관성 강제** — 같은 한국어는 시트 전체에서 같은 일본어로 (비밀번호→パスワード 통일)
3. **경어·미화어 레벨 고정** — 정중 요청형·미화어(お〜/ご〜)로 통일, 수동 직역 금지
4. **必須/任意 라벨 절대 보존** — (필수)/(선택) 라벨 추측·변경 금지, 원문-라벨 일치 자동 검증
5. **placeholder 보존 규칙** — {회원명}·{시간}·{병원명} 등 중괄호 변수는 번역 제외
6. **원문 오타 처리 규칙** — 원문 오타 자동 교정 금지, `⚠️ 원문 오타 의심` 표시 후 사람에게
7. **대안 표현 의무 + 도메인 플래그** — 채택안 1개 + 대안 2개, 불확실 시 검수 플래그

## 6. 민감정보 익명화 확인

- 골든 데이터의 서비스 브랜드명은 `서비스`/`サービス`, 고객사명은 `○○○`로 치환했다.
- 미번역 행, 내부 메모, **타사명이 잘못 들어간 행**, 신체부위·진료과목 나열 목록은 제외했다 — 범용 UX 문구만 추렸다.
- 환자·고객 실명·연락처·상담 내용·가격 정보는 캡처·포함하지 않았다.
- 보조 레퍼런스([TRIBEAU](https://tribeau.jp))에서는 공개 UI 문구만 참고했다.
- 발송·결제·로그인·외부 시스템 직접 조작은 과제 범위에서 제외했다.
- `.env`·개인 토큰 파일은 `.gitignore` 로 저장소에서 제외했다.
